# 跨境定制游DMC创业自动化执行手册

> 基于原有5人SOP（获客→客服→计调→地接→收款→执行→数据反馈）调研整理，目标：用AI agent + 结构化数据 + 低代码自动化，把团队压缩到2人。
> 调研方式：3路核心Agent + 后续20余轮补充验证，全部基于官方文档/官网直接核实，非搜索摘要。

---

## 一、总体结论

1. **没有一个现成SaaS能端到端自动化整条链路**，但用"结构化数据库 + n8n编排 + LLM"组合拳，能把大部分机械劳动自动化，把2人的精力集中在创意判断、关系维护、异常处理和资质谈判上。
2. **n8n（开源自动化编排工具）应作为整个技术栈的"胶水层"**，贯穿获客、报价、供应商协调、财务四个环节。
3. **供应商侧自动化（无论是地接协调还是支付宝对接）是行业和技术的双重"无人区"**——没有成熟产品，必须自建或走人工，这是执行时最容易被低估的部分。
4. **支付资质核实要最先做**，优先级高于所有技术搭建，因为这是business能否运转的前提。

## 二、分环节方案

### 1. 获客与客服自动化

| 环节 | 现状/工具 | 结论 |
|---|---|---|
| SaleSmartly客服 | 已有AI接待员+多语言翻译（Pro版约$12.7/月），但**无原生意向打分/结构化信息收集** | 需二次开发或换产品 |
| 客服分级方案 | **respond.io**：HOT/WARM/COLD三级打分+关键词转人工，30+语言，$79-279/月，1天上线（匹配度最高）；**SleekFlow AgentFlow**：旅游业真实案例，转化率+30%，$149-349/月 | 优先respond.io |
| 自建方案 | n8n/Dify + GPT-4o mini/Claude Haiku，月成本<$100，需2-4周技术搭建 | 预算敏感可选 |
| 广告素材生成 | HeyGen($29-149/月)/Arcads($110/月10条,约$11/条,比人工UGC便宜90%)；AdCreative.ai($39-599/月批量生成+效果打分)；Meta Advantage+ Creative(黑盒但称降本40%) | 人机协作，创意方向仍需人工把关 |
| 数据归因分析 | Northbeam($1000-2500/月)、Cometly($750+/月)、Triple Whale($129+/月)均为"建议+人工审核"，无工具原生对接SaleSmartly | 轻量自建(n8n+GPT/Gemini拉数据生成评级表)性价比最高 |

**总体判断**：客服初筛、素材初稿、数据报表可自动化70-90%，团队可压缩到1-2人负责创意终审、高意向客户跟进、异常处理；情感信任建立、创意拍板、预算最终决策仍建议保留人工。

### 2. 报价引擎（计调自动化）——最需要投入的一块

**核心结论**：Tourplan、Lemax、TravelJoy、Tourwriter等行业软件的AI功能都停留在"询价邮件结构化+行程草稿生成"，**没有现成工具能做多城市成本自动核算**，必须自建。

**自建技术路径**：
- 数据层：Airtable/Postgres价格库（酒店/用车/导游/门票/地接/交通），约200-400条核心供应商记录可覆盖80%常规路线；更新频率：酒店/机票每周，用车/导游/门票每季度
- 编排层：n8n工作流，客服表单/邮件→LLM解析结构化需求→查询价格库→按加价规则算总价→生成报价单草稿
- LLM层：Claude/GPT做function-calling，调用价格库查询工具+加价规则工具，输出结构化报价明细降低幻觉风险

**AI替代比例**：标准路线AI可生成80-90%完成度初稿，人工审核10-20分钟；复杂多城市路线仍需人工复核关键节点（城市衔接、供应商匹配合理性）。**必须保留人工**：供应商实时库存/加床确认、临时议价、行程断点经验判断、异常需求处理。

**MVP时间与成本**：约6-10周上线；自建几千到3万元人民币，外包5-15万元人民币；运维每月几百到2000元。

**现成DMC专用软件候选**（可作为"先用现成、以后再自建"的过渡方案）：

| 工具 | 定位 | 定价 | 2人团队适配度 |
|---|---|---|---|
| **Odys** | DMC专用，支持供应商实时对接(Hotelbeds等)、自动开票、盈利追踪 | $300-500/月，**不限用户数** | ★★★★★ 最推荐，按团队而非按人头收费 |
| Ezus | 功能更全（行程+供应商目录+实时利润核算+CRM） | $89-119/月/人 + 一次性搭建费€1,990-2,490 | ★★★ 前期投入较重 |
| TourCMS | 通用旅行社CMS，AI仅一个窄范围取消聊天机器人 | $49-569/月分级 | ★★ AI贡献有限 |
| Tourwriter | 行程构建+CRM，**没有真实AI功能**(官网AI相关内容是2023年展望性博客) | $99-249/月/人 | ★★ 定位偏欧美outbound |

**已排除**：Bókun/TrekkSoft/Xola/Regiondo/Rezdy/Zaui——均为"OTA分销/被动接单"平台，服务于"被中间商卖出去"，不是"主动向多城市供应商采购"的DMC工具，与核心需求不匹配。

### 3. 供应商协调自动化——技术与行业媒体的双重"无人区"

**核心结论**：没有查到任何"跨境定制游DMC供应商预订自动化"的公开真实客户案例或官方模板；行业媒体（Skift、PhocusWire）对此几乎没有专题报道。但底层技术组件（超时等待+人工兜底）是成熟的，可以自己拼装。

**关键限制（必须提前认知）**：
- **微信/企业微信不适合作为主动、批量催办供应商的渠道**：对外部联系人发消息要求对方先主动发起对话，且每天每客户限发1条群发消息。这块必须计入人员A的固定人工工作量。
- **WhatsApp Business API有24小时客服窗口**：供应商超过24小时不回复，催办消息必须走Meta预先审批的模板消息。

**推荐技术栈**：n8n自托管($5-10/月服务器) + WhatsApp Business API(Twilio/BSP，$20-50/月起) + Airtable免费版(供应商库+确认状态) + GPT-4o-mini/Claude Haiku(解析供应商回复)。**MVP预计1-2周上线，不需要外包**。n8n官方原生支持"发消息→等待确认→超时自动升级提醒人工"（WhatsApp "Send and Wait for Response"节点+Wait超时节点）。

**市场验证（真实商业产品，证明需求真实但方案偏贵/不适配）**：

| 产品 | 能力 | 适配度 |
|---|---|---|
| Lemax | 自动发邮件询价+供应商点YES/NO确认链接 | 已验证需求真实，付费黑盒 |
| Moonstride | 双向邮件确认+WhatsApp集成 | 价格不透明 |
| Travel Compositor "Group Engine"(2025) | **目前最完整**：自动发RFQ→汇总供应商回复→推荐最优组合→自动出确认单 | 值得评估，定价待确认 |
| TourConnect AI Closeouts | 解析供应商邮件自动生成预订草稿 | **已排除**：$7,500搭建费+$250/人/月，需月100+询价量，不适合2人团队 |
| Wandero AI | 声称AI原生协调供应商 | **早期/Beta**，无具名付费客户，不建议押注 |

**行业信号**：Gartner预测2027年前40%的agentic commerce项目会被取消；Deloitte指出B2B供应商侧普遍尚未准备好对接AI agent——印证"这块要自建轻量方案，不要等或买现成AI agent产品"。

### 4. 支付与财务自动化——需求最容易被误判的一块

这是调研中反复被修正的部分，以下是**最终版结论**：

**支付宝（Alipay）**：
- **在Zapier/Make/n8n中均无官方原生集成**（Zapier官方页面直接404），任何自动化都要自己写代码调用支付宝API（RSA/SHA256签名、notifyPayment异步通知）
- **不是个体商户能自助接入的**：Alipay+/Antom国际收单平台面向持牌收单机构(ACQP)，个体商户须通过第三方PSP（PingPong、连连支付、Airwallex等）间接接入，需提交营业执照、受益人信息，审核1-15个工作日
- **关键细节**：若DMC主体是**大陆注册公司**，最简单最省钱的路径其实是**直接申请支付宝官方企业商户收款码**（凭营业执照，费率有优惠），不必绕道第三方PSP

**PayPal**：
- **不支持代收支付宝**（Braintree的Alipay功能明确排除中国大陆卖家）
- **但中国主体可以直接开PayPal Business账户**（paypal.com/c2，中国专属商户页面），原生支持Online Invoicing + Request Payment
- **原生支持"一张发票分期收款"**（`allow_partial_payment`参数）：一张发票即可先收30%/40%定金、落地后再收尾款，不用分别生成两个支付链接
- 自动提醒：后台"Automatic Reminders"开关是真自动；API的remind接口需自己写调度（每天限发2次）
- 费率参考：境内卡2.99%+$0.49，ACH约1%(封顶$10)，跨境+1.5%，换汇+3-4%——**欧美客户付款实际到手大概率比预期少4-5个点**，需计入定价毛利

**目前找到的最优统一方案：Airwallex**：
- 官方文档确认**同时支持支付宝+PayPal**在同一收款流程，自带自动逾期提醒+自动对账（可同步Xero/QuickBooks/NetSuite）
- 结构细节：大陆主体通过Airwallex的**香港子实体路径**开户（大陆本身不在其"合规国家名单"），支付宝CN收款资格限定在AU/HK/SG/EU/UK主体名下
- **行动项**：开户前务必打电话跟Airwallex销售确认大陆主体能否同时开通支付宝+PayPal双通道

**免费/轻量备选**：Invoice Ninja（免费版原生支持定金+自动补尾款发票，PayPal独立网关，支付宝经Stripe网关间接支持，需符合Stripe的39国准入资格）

**已验证为假阳性、应排除的方案**：XTransfer、连连支付、PingPong——均无官方证据支持支付宝/PayPal代收（是B2B外贸打款/电商平台payout工具，与"收欧美游客付款"方向不同）；Wise Business明确不支持支付宝。

**行业对标基准**：真实同行**WindhorseTour**（中国入境游DMC）公开费率：PayPal约4.5%（不可退商户费），支付宝信用卡通道约3%，建议出发前20天内的临时预订优先用这几种——可直接作为定价时预留手续费成本的参考。

**现实建议**：
1. 支付宝：大陆主体优先直接申请官方商户收款码；同时评估Airwallex双通道方案
2. PayPal：直接开中国专属Business账户，用原生Invoicing
3. 两边对账：**订单量在几十到上百单/年的阶段，不建议追求全自动打通**，Excel/Notion按订单号人工登记交叉核对即可，等业务量起来后再考虑自动化，避免为"自动化"承担不必要的合规和接入成本
4. **这是全部方案里唯一需要"资质核实"而非单纯"技术搭建"的环节，建议列为最优先的行动项**，由人员A亲自跟进

### 5. 退改政策框架

参考WildChina、China Tour DMC、ASIA DMC、Eastbound Travelscape、G Adventures等真实同行政策整理：

| 距出发天数 | 定金/团款退还比例 | 尾款情况 | 说明 |
|---|---|---|---|
| ≥30天 | 全额退还或扣5-10%手续费 | 未收/全退 | 供应商成本压力小 |
| 15-30天 | 扣20-50%阶梯（15-21天扣30-50%，21-30天扣20%） | 部分退 | 部分酒店/用车已锁定 |
| 7-15天 | 扣80-100% | 扣30-50% | 对应原SOP"15天"分界线 |
| <7天/未出现 | 100%不退 | 100%不退 | 与国际同行一致 |
| **机票/火车票等第三方刚性资源** | 单列，**按票务方实际购买/出票规则执行**，DMC不做垫付担保 | 同左 | 这类资源DMC不可控，需在合同中单独声明 |

**执行方案**：n8n的Switch/IF节点（或更复杂场景用Camunda DMN决策表）做"算钱"层——输入取消日期/出发日期/订单金额/供应商类型，输出退款金额；LLM读取计算结果+政策文本生成客户话术草稿；**人工审核后发送**。"规则引擎算钱、LLM写话术、人工拍板"这个模式2025-2026已经成熟（Refund Evaluator、Fini Labs、Gorgias等有实际案例）。

## 三、2人团队分工

### 人员A——客户与供应商关系官（对外）
- AI客服升级的复杂/高意向/大额客户人工接待
- 地接供应商开发、谈判、关系维护（含微信/电话人工触达——这块无法自动化）
- 支付资质谈判（Airwallex/PSP开户、支付宝商户码申请）
- 广告创意方向终审
- 突发状况/客诉处理、退改政策异常case审核

### 人员B——系统与运营官（对内）
- 维护Airtable/Postgres供应商价格库、退改规则表
- 维护n8n工作流（客服分级、报价引擎、供应商协调提醒、支付催收）
- 审核AI生成的报价单初稿、退款话术
- 监控转化数据看板，调整投放
- 财务对账

## 四、实施路线图

**阶段0（立即启动，优先级最高，与其他阶段并行）**
- 核实支付资质：联系Airwallex/PingPong确认大陆主体开户可行性；同步申请支付宝官方商户码；开通PayPal Business账户(paypal.com/c2)
- 这是business能否运转的前提，必须最先验证，不要等系统都搭好才发现卡在收款这一关

**阶段1（第1-2月）**
- 落定退改政策规则表（细化到天数分级退款比例）
- 搭建Airtable供应商价格库（200-400条核心记录）
- 上线respond.io做客服分级

**阶段2（第2-3月）**
- n8n报价引擎MVP（价格库+LLM生成报价初稿）
- Invoice Ninja/PayPal Invoicing跑通定金/尾款流程

**阶段3（第3-4月）**
- n8n供应商协调工作流（WhatsApp Business API预订请求+超时提醒+Airtable状态看板）
- 广告素材生成流程（HeyGen/Arcads/AdCreative.ai）

**阶段4（业务量起来后）**
- 数据归因看板深化
- 视订单量决定是否升级到Odys等专用DMC软件，或投入自动对账系统

## 五、成本量级汇总

- 软件订阅：约¥1000-3000/月（respond.io/SleekFlow $79-349 + n8n自托管几乎免费 + LLM API + Airtable + Invoice Ninja等）
- 前期自建开发：几千到3万元人民币（若外包报价引擎/供应商协调工作流：5-15万元人民币）
- 支付手续费：预计综合成本4-5%（跨境+换汇），需计入报价毛利

## 六、风险与不确定性清单

1. **支付宝商户资质**：大陆主体能否顺利开通Airwallex双通道，或走官方商户码/第三方PSP——需主动核实，不要假设
2. **微信供应商触达无法自动化**：这是人员A的固定工作量，不要指望技术方案覆盖所有供应商
3. **退改政策目前只是行业参考框架**：需结合自己实际成本结构（供应商预付/担保条款）最终拍板，不能直接照搬
4. **报价引擎自建有真实的开发和维护成本**：不是"一次搭好就完事"，价格库需要持续更新
5. **低价引流+二次报价的话术需要人工打磨**：固化给AI客服前要设计好预期管理，避免客户被最初低价吸引后对涨价产生强烈不满
6. **供应商侧AI自动化和支付宝生态自动化都是真实的市场空白**：意味着没有教程可抄，也意味着谁先做扎实谁有先发优势
